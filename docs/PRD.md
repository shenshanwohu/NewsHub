## 1. 项目概述

### 1.1 产品定位

NewsHub CMS 是一个面向公开互联网的新闻发布系统，为多管理员团队提供从内容创作、审核到发布的完整工作流。系统面向两类用户：**访客**（浏览新闻）和**管理员**（管理内容）。

### 1.2 核心目标

1. 提供简洁、快速、SEO 友好的新闻浏览体验
2. 为管理员提供类似 Word 的富文本编辑能力
3. 支持多管理员协作与角色分级权限管理
4. 支持图片上传与管理
5. 支持全文搜索
6. 支持新闻多分类归属，增强内容组织灵活度

### 1.3 非目标（第一版范围外）

- ❌ 评论系统
- ❌ 多语言/国际化
- ❌ 定时发布
- ❌ 标签系统
- ❌ 订阅/邮件推送
- ❌ 统计分析 / 数据看板
- ❌ 第三方 CMS 数据迁移

## 2. 用户角色与权限

### 2.1 访客（未登录）

| 功能           | 说明                                             |
| -------------- | ------------------------------------------------ |
| 浏览新闻列表   | 按发布时间倒序浏览已发布新闻；置顶新闻优先展示   |
| 浏览新闻详情   | 查看完整新闻内容、多分类归属、浏览数             |
| 按分类查看新闻 | 通过分类导航过滤新闻列表（新闻在多分类下均可见） |
| 全文搜索       | 按标题和内容关键词搜索新闻                       |

### 2.2 管理员（登录用户）

采用三角色分级体系：

#### 2.2.1 超级管理员（Super Admin）

- 拥有所有权限
- 管理其他管理员账号（创建/禁用/修改角色）
- 删除任意新闻
- 管理分类（增删改）
- 上传/管理图片库
- 创建、编辑、发布、删除新闻
- 设置新闻置顶

#### 2.2.2 编辑（Editor）

- 创建新闻
- 编辑自己的草稿
- 编辑他人的草稿（经超级管理员授权后）
- 上传/管理图片库
- **无权发布新闻**
- **无权删除已发布新闻**
- **无权设置置顶**
- **无权管理分类**
- **无权管理管理员账号**

> 编辑提交的新闻需由发布者或超级管理员审核发布。

#### 2.2.3 发布者（Publisher）

- 创建新闻
- 编辑任意草稿
- 发布/下架新闻
- 删除草稿
- 上传/管理图片库
- 设置新闻置顶
- **无权删除已发布新闻**（需超级管理员）
- **无权管理分类**
- **无权管理管理员账号**

## 3. 功能需求

### 3.1 访客端（公开门户）

| 编号 | 功能模块     | 需求描述                                                                                                                                     | 优先级 |
| ---- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| F-01 | 新闻列表页   | 按发布时间倒序展示新闻卡片（标题、摘要、缩略图、发布日期、多分类标签）；置顶新闻置顶显示；支持分页                                           | P0     |
| F-02 | 新闻详情页   | 展示完整新闻内容、全部分类归属、发布时间、作者/编辑信息、浏览数；富文本内容正确渲染                                                          | P0     |
| F-03 | 分类导航     | 顶部或侧边栏显示分类列表，点击后过滤该分类下所有已发布新闻（多分类归属的新闻在各分类下均出现）                                               | P0     |
| F-04 | 全文搜索     | 输入关键词，按标题和内容匹配检索；结果按相关性排序；支持分页                                                                                 | P1     |
| F-05 | 响应式布局   | 桌面端、平板、移动端均能正常浏览                                                                                                             | P0     |
| F-06 | SEO 基础     | 每条新闻独立 URL；自定义 meta title/description（从 SEO 字段读取）；Open Graph 标签（title, description, image, type）；自动生成 sitemap.xml | P0     |
| F-07 | 置顶新闻展示 | 首页顶部展示被管理员置顶的新闻（可多条），区分于常规列表                                                                                     | P1     |
| F-08 | 浏览数显示   | 新闻详情页和列表卡片显示阅读量                                                                                                               | P2     |
| F-09 | 404 页面     | 无权限访问或页面不存在时显示友好的 404 页面                                                                                                  | P1     |

### 3.2 管理端（后台）

| 编号 | 功能模块       | 需求描述                                                                                                                              | 优先级 |
| ---- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| A-01 | 登录/登出      | 使用 Supabase Auth 邮箱密码登录；Session 管理；登录后跳转后台首页                                                                     | P0     |
| A-02 | 新闻列表管理   | 表格展示所有新闻（标题、状态、多分类标签、作者、创建时间、浏览数、置顶状态）；支持筛选（状态/分类）和排序；支持批量操作               | P0     |
| A-03 | 创建新闻       | 富文本编辑器（类似 Word）；设置标题、摘要、关联**多个分类**（多选）、封面图、SEO 信息（meta 标题/描述/OG 图片）、置顶开关；保存为草稿 | P0     |
| A-04 | 编辑新闻       | 从列表进入编辑已有新闻；可修改多分类归属、SEO 字段、置顶状态；保存草稿或提交发布                                                      | P0     |
| A-05 | 删除新闻       | 超级管理员可删除任意新闻；编辑可删除自己的草稿；二次确认                                                                              | P0     |

| A-07 | 分类管理       | 树形分类列表；增删改分类；分类可禁用                                                                                                  | P1     |
| A-08 | 图片上传与管理 | 上传图片至 Supabase Storage；图片库浏览；插入图片到富文本；复制图片链接；管理端图片懒加载                                             | P0     |
| A-09 | 管理员管理     | 超级管理员可查看管理员列表、创建新管理员、禁用管理员、修改角色                                                                        | P1     |
| A-10 | 新闻预览       | 编辑/发布者可预览新闻的最终渲染效果                                                                                                   | P1     |
| A-11 | 置顶管理       | 从新闻编辑页或列表页快速切换新闻置顶状态                                                                                              | P1     |

### 3.3 富文本编辑器要求

| 功能      | 说明                                               |
| --------- | -------------------------------------------------- |
| 文字格式  | 加粗、斜体、下划线、删除线                         |
| 段落样式  | 标题级别（H2–H4）、引用、代码块、列表（有序/无序） |
| 链接      | 插入/编辑/删除超链接                               |
| 图片      | 从图片库选择或直接 URL 插入；调整对齐方式          |
| 表格      | 插入简单表格                                       |
| 清除格式  | 一键清除选中内容格式                               |
| 源码模式  | 切换到 HTML 源码编辑                               |

> 富文本编辑器推荐方案：Tiptap（基于 ProseMirror）或 TinyMCE 自托管。

## 4. 非功能需求

| 类别     | 需求                | 说明                                                               |
| -------- | ------------------- | ------------------------------------------------------------------ |
| 性能     | 首屏加载 < 2s       | 静态生成 (SSG) + Incremental Static Regeneration (ISR)             |
| 性能     | 图片优化            | 使用 Next.js Image 自动压缩、WebP 格式、响应式尺寸                 |
| 性能     | 浏览计数策略        | 采用异步延迟写入（不阻塞页面渲染），避免每次浏览都触发数据库写操作 |
| 安全     | XSS 防护            | 富文本内容输出时进行 HTML 清洗（DOMPurify）                        |
| 安全     | CSRF 防护           | Next.js Server Actions 内置                                        |
| 安全     | 数据访问权限        | Row Level Security (RLS) 在 Supabase 层面控制                      |
| SEO      | 每条新闻独立 URL    | URL 格式：`/news/[slug]`                                           |
| SEO      | 自定义 SEO 字段     | 管理员可为每条新闻独立设置 meta title、meta description、OG image  |
| SEO      | Open Graph 支持     | og:title, og:description, og:image, og:type                        |
| SEO      | 结构化数据          | 支持 JSON-LD（NewsArticle schema）                                 |
| 可维护性 | TypeScript 严格模式 | 全量 TypeScript 代码，严格类型检查                                 |
| 可维护性 | API 设计            | Next.js App Router Server Actions 作为后端接口                     |

## 5. 数据模型概述

详见 [Database.md](./Database.md)，核心实体包括：

- **Admin/User** — 管理员账号（Supabase Auth 集成）
- **Category** — 新闻分类（树形结构，支持多级）
- **News** — 新闻文章（富文本内容，含 SEO 字段、浏览计数、置顶标记）
- **NewsCategory** — 新闻-分类关联（多对多中间表）
- **Image** — 图片元数据（Supabase Storage 引用）

> 多分类通过 `News <— NewsCategory —> Category` 多对多关系实现，一条新闻可归属零到多个分类。

## 6. 约束与假设

- 部署平台为 Vercel
- 数据库为 PostgreSQL（通过 Supabase）
- 文件存储为 Supabase Storage
- 认证系统为 Supabase Auth（邮箱密码）
- 非 SaaS 产品，不考虑多租户
- 所有管理员密码相关的登录/注册通过 Supabase Auth 完成，系统不直接存储密码
- 第一版不支持 API 对外暴露（无第三方调用场景）
- 浏览计数允许最多 5 分钟的写入延迟，不追求实时精确
