## 7. 风险与缓解

| 风险                                         | 概率 | 影响 | 缓解措施                                                            |
| -------------------------------------------- | ---- | ---- | ------------------------------------------------------------------- |
| Tiptap 集成复杂度超标                        | 中   | 高   | Phase 2 先做基础表单，Tiptap 独立开发                               |
| RLS 策略配置不当导致数据泄露                 | 低   | 高   | 每个 RLS 策略附带集成测试；`news_categories` 与 `news` 策略联动验证 |
| 富文本 XSS 绕过                              | 中   | 高   | 双重清洗（服务端 + 客户端）；CSP Header                             |
| ISR 缓存导致内容更新延迟                     | 中   | 中   | 发布时 Server Action 调用 `revalidatePath`；设置较短 revalidate     |
| Supabase 免费计划配额不足                    | 低   | 中   | 提前规划 Row Limit；图片压缩减少存储                                |
| 多分类查询性能下降（JOIN `news_categories`） | 低   | 中   | 确保 `news_categories` 索引覆盖查询模式；监控 EXPLAIN ANALYZE       |

> **v0.2 变更**：新增"多分类查询性能"风险。
