Vercel 部署方案

前提条件

- 代码已推送到 GitHub：github.com/shenshanwohu/NewsHub
- Supabase 新加坡项目已就绪（已配好）
- .env.local 里有全部环境变量

部署步骤

1. 打开 Vercel 并导入仓库

浏览器打开 https://vercel.com/new ，用 GitHub 登录后：

- 点击 Import Git Repository
- 找到 shenshanwohu/NewsHub
- 点 Import

2. 配置环境变量

在 Vercel 的配置页面里，找到 Environment Variables 部分，添加以下 4 个变量（从你的
.env.local 复制值）：

┌───────────────────────────────┬────────────────────┐
│ 变量名 │ 值来源 │
├───────────────────────────────┼────────────────────┤
│ NEXT_PUBLIC_SUPABASE_URL │ 从 .env.local 复制 │
├───────────────────────────────┼────────────────────┤
│ NEXT_PUBLIC_SUPABASE_ANON_KEY │ 从 .env.local 复制 │
├───────────────────────────────┼────────────────────┤
│ SUPABASE_SERVICE_ROLE_KEY │ 从 .env.local 复制 │
├───────────────────────────────┼────────────────────┤
│ DATABASE_URL │ 从 .env.local 复制 │
└───────────────────────────────┴────────────────────┘

还有一个需要填的：

| NEXT_PUBLIC_SITE_URL | https://你的项目名.vercel.app（部署完成后会看到） |

可以先填临时占位，部署成功后拿到域名再改。

3. 部署设置（保持默认）

- Framework Preset: Next.js（自动检测）
- Build Command: next build（默认）
- Output Directory: .next（默认）
- 不用改任何东西，直接点 Deploy

4. 等待部署完成

大约 1-2 分钟。完成后会显示一个域名 https://news-hub-xxx.vercel.app。

5. 部署后补充操作

- 拿到域名后，去 .env.local 更新 NEXT_PUBLIC_SITE_URL（会让 sitemap 和 robots.txt
  用上正确域名）
- 去 Supabase Dashboard →Authentication →Settings，把 Site URL 改为 Vercel
  的域名（确保登录后能正确跳转）

注意事项

- Supabase Auth 回调：登录功能需要在 Supabase 后台配一下允许的域名
- 打开 https://supabase.com/dashboard/project/你的项目/auth/settings
- 在 Site URL 填 https://你的项目名.vercel.app
- 在 Redirect URLs 添加 https://你的项目名.vercel.app/\*\*
